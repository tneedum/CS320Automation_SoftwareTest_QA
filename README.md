# CS320Automation_SoftwareTest_QA

In this course, I learned how to ensure that my code and programs are both functional and secure by using unit testing and validation techniques. For each part of Project One, I created JUnit tests to verify that the contact, task, and appointment services met the required constraints. This included testing for invalid inputs such as null values, overly long fields, and incorrect data. By using assertions and exception handling, I was able to confirm that the program behaved correctly under both normal and edge case conditions. This approach helped improve the overall reliability and security of the software.
To interpret user needs, I focused on understanding the requirements provided for each service and translating them into specific validation rules and test cases. For example, if a requirement stated that a contact ID must not be null or exceed a certain length, I created test cases to enforce those rules. This ensured that the program aligned with what the user expected and prevented invalid data from being stored or processed.
When designing software, I now take a more structured approach by breaking the program into smaller components and testing each part individually. I used object-oriented principles to separate responsibilities across different classes, such as having separate service and test classes. This made the code easier to manage, test, and update. Overall, this course helped me improve my ability to design programs that are reliable, secure, and maintainable through proper testing and organization.
